# Security Policy

EVE is designed as a local-first AI workflow project.

## Reporting a Security Issue

If you discover a security issue, please do not open a public GitHub issue.

Instead, contact Evercrew Venture Pte Ltd at:

support@evercrew.ai

Please include:

- Description of the issue
- Steps to reproduce
- Possible impact
- Suggested fix, if any

## Security Principles

EVE follows these principles:

- Local-first deployment
- No hardcoded secrets
- Environment variables for credentials
- Human confirmation before task assignment
- Minimal external dependencies
- No cloud AI required by default

## Important Reminder

Do not commit your `.env` file or Telegram bot token to GitHub.
