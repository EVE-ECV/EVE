# Contributing to EVE

Thank you for your interest in contributing to EVE.

We welcome developers, students, SME owners, AI enthusiasts, and anyone interested in practical AI workflow automation.

## Before You Start

Please read:

- README.md
- docs/architecture.md
- docs/philosophy.md
- docs/roadmap.md

Understanding the project vision helps keep contributions aligned.

---

## Our Goals

EVE is built around several core principles:

- Practical AI
- Local-first deployment
- Human approval before automation
- Simple architecture
- Beginner-friendly code
- Enterprise-ready design

---

## Coding Guidelines

- One file should have one responsibility.
- Write clear and readable code.
- Keep functions focused on a single purpose.
- Add docstrings for public functions.
- Comment why something exists, not only what it does.

---

## Reporting Issues

Please include:

- Operating system
- Python version
- Ollama version
- Error messages
- Steps to reproduce

---

## Feature Requests

Before requesting a feature, ask:

- Does it solve a real SME problem?
- Can it remain simple?
- Can it work locally?
